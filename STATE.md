# GuPiao Loop 状态

最后更新：2026-07-24

## 当前目标

交付八套仅模拟盘的独立量化策略。每套策略使用独立的 200 万元账户、任务、持仓、订单、
风控和绩效；八套合计虚拟本金 1,600 万元，并保留原一夜持股法、概率组合和
TradingAgents 的账户与历史。

## 本轮结果

- 功能分支为 `codex/independent-strategy-suite`，位于专用 worktree
  `.worktrees/independent-strategy-suite`，未直接修改 `main`。
- 已实现多因子核心、相对强弱轮动、突破趋势、短期反转 T+1、低波质量、业绩公告漂移、
  市场状态配置和风险平价组合八个独立模块。
- 已实现真实点时日线、复权、估值、财务、公告、ETF 和基准数据存储与同步；关键字段缺失、
  演示数据、未来数据、过期行情或数据批次不完整时只暂停受影响策略。
- 已实现 16:30 至 16:37 分散信号、下一交易日 09:35 至 09:42 分散执行、数据库任务租约、
  崩溃恢复、幂等键和单策略错误隔离。
- 已实现完整目标组合预检、先卖后买、T+1、整手、现金、费用、滑点、仓位、日亏损、回撤、
  连续错误和紧急停止，并保存候选、决策快照、订单、绩效和风控事件。
- 已实现 500 个交易日回测、98% 数据完整率、样本外收益、Sharpe、回撤、交易次数和当前配置
  无下单演练闸门；自动计划默认关闭。
- 已新增管理员 API、八策略横向比较和详情控制台，以及 SQLite 运行时迁移、MySQL `0004`
  增量迁移、`quant-strategy-worker`、Docker Compose 和 LaunchAgent 入口。
- 已实现 AKShare 与 Tushare 证券主数据合并，以及 Tushare 日线、复权、每日指标和财务报表的
  横截面批量同步；历史不足的冷启动股票仍按单股回补，财务 VIP 权限不足时对应策略失败关闭。
- `quant-strategy-worker` 使用八条策略实时任务通道和两条独立回测通道；信号、执行和数据同步
  在各自恢复窗口内幂等重试，普通计划接口不能修改或删除量化策略内部计划。
- 隔离运行发现并修复了多进程同时种子化导致唯一键冲突和重复账户的风险。现在由 API 后端
  单独初始化，Worker 等待后端健康后启动。
