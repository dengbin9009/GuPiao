# GuPiao Loop 状态

最后更新：2026-07-14

## 当前目标

保持既有一夜持股法模拟链路稳定运行，并交付独立的 TradingAgents 全自动模拟组合策略。在不接触实盘、不输出密钥、保持 `LIVE_TRADING_ENABLED=false` 与 `BROKER_ADAPTER=simulation` 的前提下，让后端、前端、通用 Worker、调度器和 TradingAgents Worker 持续运行。

## 本轮结果

- 初始基线：后端 `58 passed / 3 failed`，scheduler 因交易日历提供方全部失败而退出。
- 当前后端：Python 3.12.12 下全套测试 `177 passed`；Ruff 全部通过；wheel 已从干净构建环境生成；全新环境可一次安装 `market + agents + dev` 依赖。
- 当前前端：Node 20 下生产构建通过，共转换 1560 个模块。
- TradingAgents：新增独立策略、10 万元独立模拟账户、13:25 快照、Top 100/Top 10 确定性预筛、五级评级、三种仓位映射、批次租约、预算与截止时间、原子模拟调仓、管理员 API 和控制台。上游固定为 `v0.3.1` / `01477f9`，readiness 同时校验版本和 Git 提交。
- 快照审计：GuPiao 行情、日线、指标和公告为核心依据；Yahoo 基本面与新闻只在 LLM 调用前作为可选数据冻结进 SHA-256 快照，剔除价格派生字段，抓取失败记录不可用哨兵。分析阶段不临场访问任何外部数据工具。
- 账户隔离：TradingAgents 只能绑定未被其他策略配置占用的活跃模拟账户；一夜持股法配置显式绑定默认账户。账户总资产统一按现金加全部持仓最新市值重估。
- AI 安全门：无 `OPENAI_API_KEY` 时 readiness 只报告未配置，不输出密钥；`dry_run=true`，`agent_analysis` 与 `agent_rebalance` 均关闭。子进程在项目树外的临时目录运行，不能绕过环境白名单自动读取项目 `.env`。
- 本地运行：LaunchAgent 托管的五个进程连续运行超过 5 分钟，后端健康端点与前端入口均返回 HTTP 200；监督日志记录 60 秒验收通过。
- Docker：`docker compose config` 通过；实际镜像构建在拉取 `python:3.12-slim` 前被本机 `dockerproxy.com` 镜像证书错误拦截，属于 Docker Desktop 外部配置，尚未完成容器持续运行验收。
- scheduler：计划使用数据库条件更新进行原子占用，并带 45 秒可恢复租约；瞬态数据失败按 15 秒退避，崩溃后会对账已有运行，避免重复下单。入场只允许 14:40 后 59 秒内执行，退出可在 09:35 至 10:00 内恢复执行。
- 一键启动：幂等准备 10 万模拟账户、观察股票、一夜持股策略和两个启用计划；要求环境保持模拟模式，并主动关闭数据库 LIVE 标志、LIVE 账户和 LIVE 计划；管理员密码会与当前安全配置同步，启动失败会清理本轮进程。
- 持续运行：macOS 用户级 LaunchAgent `com.gupiao.simulation-loop` 已启用，外层 `caffeinate` 防止空闲睡眠，五个子进程异常退出时由 launchd 重启。
- 数据真实性：AKShare 已同步 5530 只股票主数据；mootdx 实际刷新 5203 只沪深行情；AKShare 公告源实际保存 9059 条记录。观察版行情超过 60 秒、公司事件超过 1800 秒、时间在未来或来源不健康时均禁止下单。
- 自动窗口：公告在交易日 14:20-14:35 每 300 秒同步；全市场行情在 14:35-14:41 刷新；09:30-10:00 只刷新实际模拟持仓，避免全市场轮询拖慢退出。通知发送在交易数据窗口外处理。
- 模拟交易：一夜持股法模拟配置已启用，入场计划 14:40、退出计划 09:35；端到端测试证明到点只创建一笔模拟订单，同一窗口不重复，T+1 次交易日释放后才可卖出，不同策略配置只卖出自身历史成交归属的仓位。
- 数据库：本地 SQLite 已启用 WAL、30 秒 busy timeout 和旧表 `next_run_at` 自动迁移；worker 在提交失败后先 rollback，主循环捕获异常后继续运行。
- LIVE 硬门：API 写入口和最终下单函数都要求环境层两把开关同时开放；当前运行配置关闭，实际调用启用 LIVE 接口返回 HTTP 403，数据库 LIVE 关闭、紧急停止开启、启用账户/计划/真实订单均为 0。
- 60m 回测：`000001.SZ`，2026-06-24 至 2026-06-25，`timeframe_used=60m`，`data_source=cache`，尾盘最后一根 K 线入场，净收益 `-24.1861`，收益率 `-0.2419%`，退出码 0。
- 安全状态：数据库 LIVE 关闭，真实订单数为 0，运行配置保持 `LIVE_TRADING_ENABLED=false`、`BROKER_ADAPTER=simulation`。
- 当前服务由 LaunchAgent 持续托管；是否产生模拟订单仍取决于交易日、数据新鲜度、策略候选和全量风控预检。
- 当前限制：换手率、VWAP、MA5 与真实上市日期仍是 v1 占位字段；当前自动选股主要使用真实价格、涨跌幅、成交额、交易状态和公告风险。后续应作为独立增量 spec 补齐，不应把当前结果解读为完整因子模型。

## 观察项

- 外部行情、公告与交易日历提供方可能不稳定；验收已覆盖失败降级，到点仍可能因外部源不可用而不下单。
- 周末不会交易；服务运行不等于保证产生订单，策略没有候选或风控拦截同样是正确结果。
- 60m 回测结果为单次历史结果，不构成收益承诺。
- 外部行情或公司事件数据恢复前，策略不会创建模拟订单；这属于数据风控失败关闭，不是进程故障。
