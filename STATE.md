# GuPiao Loop 状态

最后更新：2026-07-23

## 当前目标

在保留原「一夜持股法」和 TradingAgents 模拟策略的前提下，交付并行的
`一夜持股概率组合`：独立 200 万元模拟账户，每日最多选择 10 只股票，按校准盈利概率、
预期净收益和 20 日波动率非等权分配，单股仓位 2% 至 36%，组合总仓位 30% 至 60%，
14:40 入场，下一交易日 10:30 退出并重试至 10:45。

## 本轮结果

- 功能分支：`codex/overnight-probability-portfolio`；专用 worktree：
  `.worktrees/overnight-probability-portfolio`。没有在 `main` 直接修改。
- 新增策略定义、独立账户、模型产物、训练样本、组合运行、逐股决策和策略持仓批次实体，
  同时提供 SQLite 运行时迁移和 MySQL 增量迁移。
- Worker 在交易日 14:40 自动保存无下单观察，下一实际交易日 10:30 生成净收益标签并尝试训练；
  自动训练不会自行开启交易计划。
- 模型固定使用按时间切分的逻辑回归和单调校准。训练样本至少 500 条、校准样本至少
  100 条、Brier 分数不高于 0.25；最新模型失败时不会回退旧模型交易。
- 选股范围为沪深两市成交额 Top 100，不含北交所；真实上市日期、流通股本、换手率、
  VWAP、尾盘收益、日线、基准和公告任一关键数据缺失或过期都会逐股失败关闭。
- 组合最多持有 10 只，合格不足 10 只仍按实际数量交易；校准盈利概率至少 55%、预期净收益
  必须大于 0。风险调整分数为概率优势乘预期净收益再除以 20 日波动率。
- 资金按含滑点价格、佣金、过户费和可用现金反推 100 股整数手数量；执行层硬限制单股
  2% 至 36%、总仓位不超过 60%，并防止通过异常数据库参数放宽行情、公告和日亏损安全门。
- 每笔买入创建策略归属批次；10:30 退出只处理本策略、已满足 T+1 的持仓。行情异常时
  每 15 秒重试至 10:45，未成交持仓保留并在后续交易日继续退出。
- 正式入场前必须存在可用模型、启用的 10:30 退出计划和同配置、同模型、14:40 窗口的
  管理员专用无下单演练。自动观察不能解锁交易，配置或模型变化会使旧演练失效。
- 修复调度崩溃恢复竞争：同一 14:40 窗口内的训练观察不会被误认成正式入场；历史残留的
  错误 `last_run_id` 也会按触发类型重新核对。
- 管理控制台已增加概率组合状态、模型 readiness、配置、计划、运行详情、候选概率、目标仓位
  和拒绝原因；专用 API 仅允许管理员和模拟盘。

## 当前运行状态

- 当前主目录 `main` 和现有 `127.0.0.1:5173/8000` 服务没有被本轮替换或重启，因此正在运行的
  服务尚不包含本分支新增的概率组合策略。
- 本轮没有 push、merge、deploy 或修改主服务；这些动作仍处于人工门之后。
- 即使后续合并并重启，概率组合的两条自动计划也默认关闭，不会立刻产生模拟订单。

## 启用顺序

1. 人工批准合并功能分支并重启主目录五个服务。
2. 由 Worker 在真实交易窗口积累完整训练标签；按 80/20 时间切分通常至少需要 625 条总样本。
3. readiness 确认最新模型满足 500 条训练、100 条校准和 Brier 不高于 0.25。
4. 管理员在 14:40 使用当前配置和当前模型完成一次数据完整的专用无下单演练。
5. 先人工启用 10:30 退出计划，再人工启用 14:40 入场计划。

## 仍需关注

- 外部行情、日线、公告和交易日历提供方可能不稳定；数据不完整时不下单是预期的安全行为。
- 新策略尚未积累真实模型样本，因此当前只能诊断演练，不能把因子分数或固定常量冒充盈利概率。
- Docker 五进程镜像构建仍受本机 Docker Desktop 镜像代理证书问题影响，本轮未改变该外部配置。
- 原一夜持股法和 TradingAgents 的账户、配置、历史记录及现有运行状态均未由本轮修改。
