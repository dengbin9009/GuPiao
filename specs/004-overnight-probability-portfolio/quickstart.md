# 快速开始：一夜持股概率组合策略

## 安全前提

- Python 使用 3.12，Node.js 使用 20。
- 保持 `LIVE_TRADING_ENABLED=false`。
- 保持 `BROKER_ADAPTER=simulation`。
- 不需要也不允许配置真实券商账户。

## 初始化

后端启动时会幂等创建：

- `一夜持股概率组合` 策略定义。
- `一夜持股概率组合模拟账户`，默认本金 2,000,000 元。
- `14:40` 的组合入场计划，默认关闭。
- `10:30` 的组合退出计划，默认关闭。

已有同名账户和策略不会被重置余额或覆盖配置。

## 训练模型

```bash
cd backend
.venv/bin/python scripts/train_probability_portfolio.py --through 2026-07-22
```

训练命令只生成模型产物和 readiness 报告，不启用计划、不创建订单。需要至少500条有效训练样本、100条独立校准样本且Brier分数不高于0.25。

## 无下单演练

在管理界面打开策略中心，选择「一夜持股概率组合」并执行无下单演练。演练会保存候选、概率、预期收益和目标仓位，但订单数必须为0。

只有模型和真实因子都就绪、且最新配置完成一次成功演练后，管理员才可启用14:40入场计划。10:30退出计划可独立启用以处理已有模拟持仓。

## 默认资金规则

- 每日0至10只股票。
- 校准盈利概率至少55%，预期净收益大于0。
- 单股仓位2%至36%。
- 总仓位30%至60%，剩余资金保留现金。
- 次日10:30退出；行情异常时重试至10:45。

## 验证

cd backend
