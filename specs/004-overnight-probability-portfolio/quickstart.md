# 快速开始：一夜持股概率组合策略

## 当前落地状态

- 功能已在 `codex/overnight-probability-portfolio` 分支实现，尚未自动合并、推送或重启主目录服务。
- 独立 200 万元模拟账户会在新代码首次启动时幂等建立，两条自动计划默认关闭。
- 当前分支已通过 Python 3.12 后端全套测试、Ruff、Node 20 前端测试与构建，以及隔离数据库五进程持续运行验证。
- 在完成模型样本积累、管理员 14:40 专用无下单演练和计划人工启用前，本策略不会自动创建模拟订单。

## 安全前提

- Python 使用 3.12，Node.js 使用 20。
- 保持 `LIVE_TRADING_ENABLED=false`。
- 保持 `BROKER_ADAPTER=simulation`。
- 不需要也不允许配置真实券商账户。

## 初始化

后端启动时会幂等创建：

- `一夜持股概率组合` 策略定义。
- `一夜持股概率组合模拟账户`，默认本金 2,000,000 元。
- `14:40` 的组合入场计划，默认关闭。
- `10:30` 的组合退出计划，默认关闭。

已有同名账户和策略不会被重置余额或覆盖配置。

## 训练模型

Worker 会在每个交易日 14:40 自动保存一批无下单训练观察，并在下一实际交易日
10:30 使用新鲜行情生成净收益标签。标签完成后会自动训练一个新模型产物，但不会
自动启用任何交易计划。

逻辑回归使用前 80% 样本训练、后 20% 样本校准。默认门槛分别是 500 条训练样本和
100 条校准样本，因此在 80/20 切分下，通常至少需要 625 条完整总样本。若 Top 100
候选的数据质量完整，系统可在多个交易日内逐步积累；缺失真实因子的股票不会进入样本。

也可以由管理员对已有样本执行显式训练：

```bash
cd backend
.venv/bin/python scripts/train_probability_portfolio.py --through 2026-07-22
```

训练命令只生成模型产物和 readiness 报告，不启用计划、不创建订单。最新训练产物若
校准失败，系统会保持模型未就绪，不会退回更旧的模型继续交易。

## 无下单演练

在管理界面打开策略中心，选择「一夜持股概率组合」并执行无下单演练。模型未就绪时
也可以执行诊断演练，查看真实因子缺口；演练会保存候选、概率、预期收益和目标仓位，
订单数始终为 0。

只有当前最新模型和真实因子都就绪、且当前配置在 14:40 决策窗口完成一次数据完整的
专用演练后，管理员才可启用自动交易。启用顺序固定为：

1. 先启用 10:30 退出计划。
2. 再启用 14:40 入场计划。

关闭退出计划会同步关闭入场计划。配置改变或产生新模型后，旧演练失效，需要重新演练。

## 默认资金规则

- 每日0至10只股票。
- 校准盈利概率至少55%，预期净收益大于0。
- 单股仓位2%至36%。
- 总仓位30%至60%，剩余资金保留现金。
- 次日10:30退出；行情异常时重试至10:45。

## 数据范围与失败处理

- 只扫描上海、深圳交易所股票，不包含北交所。
- 每日先按成交额稳定排序取 Top 100，再执行真实因子、事件和模型过滤。
- 上市日期和流通股本会缓存；同一交易日按最新累计成交量重算换手率。
- 最新日线必须覆盖最近预期交易日；节假日可由当日数据源重新确认缓存，网络失败时不会使用未经确认的旧日线继续交易。
- 日线、分钟线、财务或公告缺失时逐股拒绝，不使用演示值或常量补齐。
- 训练样本按完整 14:40 批次切分，不能把同一决策时点横跨训练集与校准集；标签使用入场时冻结的手数、佣金、税费和滑点。
- 上一批策略持仓尚未退出时禁止叠加新仓；退出失败的持仓在后续交易日继续处理。
- 正式买入执行器会再次确认 10:30 退出计划已启用，即使绕过 API 也不能形成只买不卖的自动配置。
- Worker 自动训练观察不能解锁自动交易，只有管理员专用演练可以解锁。
