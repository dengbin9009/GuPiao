# 实施计划：八套量化策略独立运营平台

## 架构

采用“共享点时数据仓库、独立策略插件、数据库任务队列、统一模拟组合执行器”的结构：

1. Worker 在 16:15 增量同步证券主数据、日线、复权、估值、财务和公告。
2. Scheduler 在 16:30 至 16:37 为八套策略分别创建信号任务。
3. `quant-strategy-worker` 通过原子领取和可恢复租约执行任务并保存组合决策。
4. 下一交易日 09:35 至 09:42 分散创建执行任务，模拟执行器按独立账户调仓。

## 核心设计

- `StrategySpec` 定义策略键、资产类型、周期、默认参数、所需数据和信号构建函数。
- 每套策略实现独立模块，输出统一的 `TargetPortfolio`，不直接创建订单。
- `QuantStrategyTask` 提供幂等键、状态、租约、错误计数和截止时间。
- `quant-strategy-worker` 在单进程内为八套策略各建立一条信号/执行通道，并使用两条独立回测通道；长回测或单策略卡顿不得占住其他策略的交易任务。
- 成熟证券的日线、复权因子和每日指标按交易日使用 Tushare 全市场横截面批量增量同步；首次进入候选池或历史不足 520 根时才逐股回填历史。
- 点时财务优先使用 Tushare `*_vip` 按报告期批量同步。权限不足时 `quant_financial` 失败关闭，只阻塞依赖财务的策略，不退化为大规模逐股请求。
- `QuantPortfolioDecision` 与 `QuantCandidateScore` 追加保存可审计决策。
- `StrategyRiskProfile` 和 `StrategyPerformanceDaily` 按策略配置和账户隔离。
- 运行时迁移同时覆盖 SQLite；MySQL 使用新的增量 SQL，禁止删除或覆盖历史表。

## 交付顺序

1. 中文 SpecKit 文档与契约。
2. 数据模型、策略注册表、默认账户和计划。
3. 八套策略纯函数及无未来数据测试。
4. 就绪状态、回测闸门、任务租约和组合执行器。
5. 管理员 API、控制台和通知。
6. 第六进程部署、全套测试与安全验收。
