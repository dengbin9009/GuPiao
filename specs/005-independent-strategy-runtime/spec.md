# 功能规格：八套量化策略独立运营平台

**功能分支**：`codex/independent-strategy-suite`
**状态**：已实现，等待真实数据回测与逐套演练
**适用模式**：仅 `SIMULATION`

## 目标

在不改变一夜持股法、概率组合和 TradingAgents 历史账户的前提下，增加八套独立量化策略。每套策略拥有独立的 200 万元模拟账户、配置、任务、组合决策、订单、持仓、风控和绩效，八套之间只共享可信市场数据与通用执行基础设施。

## 用户故事与验收

### 独立运行

1. 系统启动时幂等注册八套策略、八个独立账户、信号计划和执行计划。
2. 已存在的同名账户不得被重置余额、持仓或历史记录。
3. 一个账户最多绑定一套独立策略；订单、持仓、账本和绩效只能归属于绑定账户。
4. 某套策略数据缺失、超时、熔断或暂停，不得阻塞其他策略任务。

### 可信数据与决策

1. 股票和 ETF 都使用统一证券主数据，记录证券类型、每手数量和结算周期。
2. 策略只能读取决策时刻已经公开的日线、复权因子、估值和财务数据。
3. 只有公告日期而没有时间的财务记录，从下一交易日开始可用。
4. 每次决策保存规范 JSON 快照和 SHA-256，包含数据截止时间、来源、候选、特征、目标仓位和拒绝原因。
5. 核心字段缺失、数据位于未来或演示数据混入时，策略状态为 `DATA_PENDING`，订单数必须为零。

### 上线闸门

运营状态固定为：

- `DATA_PENDING`：必需数据不完整。
- `BACKTEST_PENDING`：尚无合格回测。
- `DRY_RUN_PENDING`：回测合格但当前配置尚未演练。
- `READY`：满足数据、回测和演练门槛，等待管理员启用。
- `ACTIVE`：模拟调度已启用。
- `PAUSED`：管理员或风控暂停。
- `FAILED`：连续错误或不可恢复任务失败。

合格回测必须使用至少 500 个交易日、数据完整率不低于 98%、样本外年化收益大于零、Sharpe 不低于 0.3、最大回撤不高于 25%，且至少包含 30 次交易或调仓。配置、策略版本或数据版本变化后，旧演练失效。

### 模拟调仓与风控

1. 信号在收盘后产生，模拟执行在下一交易日使用新鲜实时报价。
2. 执行器先构建完整目标组合并全量预检，再在一个数据库事务中先卖后买。
3. 统一遵守现金、100 股整数手、T+1、佣金万三且最低 5 元、卖出印花税万五和 5bp 滑点。
4. 每套策略独立设置日亏损 2%、最大回撤 15%、连续错误 3 次、最大订单数和紧急停止。
5. 所有新策略硬限制为模拟盘；实盘开关、适配器或账户状态异常时失败关闭。

## 成功标准

- 八套策略可分别完成数据就绪检查、真实数据回测、无下单演练、人工计划启停和模拟执行。
- 控制台可横向比较八个账户的总资产、收益、回撤、持仓、下一任务和错误。
- 单策略故障不会产生重复订单，也不会改变其他策略账户。
- 自动计划默认关闭，真实订单和启用真实账户始终为零。
