# 技术研究：八套独立策略平台

## 决策

- 采用业务隔离而不是复制八套服务。账户、任务、风控和绩效独立，行情、数据库连接和执行算法共享。
- 动量、反转和盈利能力定义参考 Kenneth French 数据库及经典因子研究；参数在 A 股点时数据上回测后才能进入 `READY`。
- 业绩漂移只使用实际公告时点可见的财务数据，不用当前修订值回填历史。
- 风险平价使用等风险贡献迭代求解和收缩协方差；不加杠杆，目标波动只能缩减总仓位。
- 公开数据权限不足时单策略失败关闭，不以估算常量替代 ROE、毛利率、经营现金流或复权因子。

## 数据来源

- Tushare：复权因子、每日指标、财务指标、利润表、现金流量表、资产负债表、ETF 基础和日线。
- AKShare：股票/ETF 日线和主数据后备。
- mootdx：下一交易日模拟执行所需的新鲜报价。
- 交易所公告/现有公告路由：停复牌、风险和财务公告时点。

## 同步吞吐

- Tushare 官方接口支持 `daily`、`adj_factor`、`daily_basic` 按 `trade_date` 获取全市场横截面，日常增量固定使用该方式，避免循环 800 个股票请求。
- 历史不足 520 根的证券仍按代码补齐约 1,200 个自然日数据，达到基线后切换为横截面增量。
- 财务普通接口只能逐股获取；系统优先使用 `fina_indicator_vip`、`income_vip`、`cashflow_vip`、`balancesheet_vip` 按报告期批量获取。缺少对应权限时财务数据集保持未就绪。
- Worker 采用八条实时策略通道和两条回测通道，继续作为一个部署进程运行，兼顾业务隔离与部署复杂度。

## 兼容性

- Python 固定 3.12，避免引入需要单独运行时的新技术栈。
- 统计公式使用标准库实现，保持安装轻量；数据同步可继续使用 pandas 可选依赖。
- SQLite 用于本地运行和测试，Linux Docker Compose 使用 MySQL 8.4。
