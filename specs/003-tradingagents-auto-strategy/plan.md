# 实施计划：TradingAgents 全自动模拟组合策略

## 架构

采用“主系统编排、独立 Worker 分析、模拟事务调仓”的三段式架构：

1. 通用 Worker 在 13:25 固化全市场实时行情、Top 100 加持仓的日线和公告。
2. 调度器在 13:30 幂等创建批次；独立 `tradingagents-worker` 原子领取批次，并发启动每股子进程。
3. 调度器在 14:45 消费 `ready` 批次，完成全量风控预检和原子模拟调仓。

## 关键决策

- TradingAgents 固定 `v0.3.1` / `01477f9`，作为可选 `agents` 依赖安装。
- 每股研究运行在独立子进程，不进入现有 5 秒插件沙箱。
- 子进程只接收固化快照和必要模型配置；环境只白名单传递 OpenAI 与网络变量，并在项目树外的独立临时目录运行，阻止上游自动加载项目 `.env`。
- 数据快照使用规范 JSON 与 SHA-256，可审计、可回放。
- 自动调度采用失败关闭：密钥、依赖、干跑、模拟盘隔离任一缺失均不能启用。
- 原有一夜持股法保持原账户和原执行入口，新增策略通过注册表按 key 分派。

## 交付阶段

1. 数据模型与运行时迁移。
2. 独立账户、策略定义与默认关闭计划。
3. 行情固化、确定性预筛与快照门禁。
4. 上游适配器、子进程、并发、预算与租约。
5. 跨股票决策、三种仓位映射与原子调仓。
6. API、控制台、五进程部署与中文文档。
7. 全套测试、Ruff、前端构建、Docker 配置和持续运行验证。
